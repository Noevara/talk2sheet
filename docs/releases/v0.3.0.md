# Talk2Sheet v0.3.0

Talk2Sheet `v0.3.0` is a workbook routing and multi-sheet usability release.

This release keeps execution safe and explicit: each turn still runs on one sheet, while multi-sheet questions are detected, clarified, and decomposed into sequential steps.

## Highlights

- Workbook-level multi-sheet detection and clarification flow
- Sequential multi-sheet follow-up routing (analyze sheet A first, then continue on sheet B)
- User-visible routing explanation fields in answer cards:
  - `reason`
  - `explanation`
  - `explanation_code`
- Workbook overview cards with per-sheet:
  - row/column counts
  - field summary
- Expanded intent regression corpus to `v0.3.0` scenarios:
  - single-sheet
  - multi-sheet clarification
  - multi-sheet sequence
  - multi-sheet pronoun follow-up
- Added lightweight multi-sheet observability in pipeline/logs:
  - `multi_sheet_detected`
  - `clarification_sheet_count`
  - `sheet_switch_count`
  - `multi_sheet_failure_reason`
  - `multi_sheet_top_failure_reasons`

## Included In This Release

### Workbook routing and follow-up semantics

- Added boundary detection for questions that mention multiple sheets
- Added decomposition hints when a question should be split into sequential sheet-level analysis
- Added follow-up sheet-switch handling for:
  - explicit sheet switch
  - “another sheet”
  - “previous sheet”
- Kept single-turn execution bounded to one sheet

### Frontend capability visibility

- Added workbook overview UI to help users choose the next sheet by context
- Added source-sheet/switch metadata on conversation result cards
- Added routing boundary and explanation rendering in chat cards
- Refreshed categorized example prompts to better reflect supported workflows and boundaries

### Quality and governance

- Upgraded intent regression fixture and evaluator to `v0.3.0`
- Added scenario-level breakdown in offline evaluator output
- Added/updated regression tests for:
  - sheet router boundary decisions
  - follow-up switch behavior
  - workbook metadata summary loading
  - routing visibility in frontend cards

## Scope Reminder

Supported in `v0.3.0`:

- Workbook-aware routing with one-sheet execution per turn
- Multi-sheet clarification and sequential A→B analysis guidance
- Compare/ranking/share/detail/trend/basic chart/lightweight forecast
- Clarification and multi-turn follow-up conversation
- Request-level and routing-level observability for troubleshooting

Not included in `v0.3.0`:

- Cross-sheet joins in one step
- Unrestricted combined multi-sheet execution
- Advanced statistics and causal inference
- Production-ready object storage and persistent session backends
