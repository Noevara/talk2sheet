# Talk2Sheet v0.1.1

Talk2Sheet `v0.1.1` is a usability and stability update on top of the first public open-source release.

This release does not expand the product boundary beyond workbook-aware single-sheet analysis. Instead, it makes the current scope easier to try, easier to debug, and easier to keep using across refreshes and local restarts.

## Highlights

- One-click copy for assistant answer cards
- CSV export for result tables
- PNG export for result charts
- Local session restore for the latest workbook, sheet, mode, and conversation state
- Clearer clarification UX for both column and sheet disambiguation
- Clearer upload, preview, and conversation error handling
- Better request-level observability with `X-Request-ID` across frontend and backend
- More explicit Docker / `.env` guidance for local LLM-backed startup
- Additional frontend and backend regression coverage
- Small engineering cleanup to remove stale compatibility drift and reduce UI wiring overhead

## Included In This Release

### User-visible improvements

- Copy the final structured answer directly from the result card
- Export detail or result tables as CSV
- Export charts as PNG directly from the chart card
- Refresh the page and recover the latest local workbook context when the uploaded file is still available
- Clarification cards now distinguish column vs sheet routing with typed labels and visual style
- Clarification prompts now use a friendlier reason prefix, and selected options are sent back as natural confirmation text before continuing the same question
- See clearer guidance for:
  - unsupported file type
  - oversized uploads
  - preview failures
  - streaming connection failures
  - expired clarification context
  - expired restored workbook state

### Observability and debugging

- Frontend requests now generate and send `X-Request-ID`
- Backend HTTP middleware logs request start / completion with the same id
- Upload, preview, and SSE chat entry routes now emit route-level structured events
- Architecture docs now include a concrete `request_id` troubleshooting workflow

### Engineering cleanup

- Persisted app-session logic moved out of `AppShell` into a dedicated composable
- Conversation message rendering now consumes `UiMessages` directly instead of a large manually mapped labels object
- Backend test imports were aligned with canonical planner and exact-execution modules
- Full backend test collection and execution are green again

## Scope Reminder

Supported in `v0.1.1`:

- Workbook-aware single-sheet routing and analysis
- Clarification and follow-up conversation
- Summary / ranking / detail rows / trend / basic chart / lightweight forecast
- Structured pipeline visibility and request-level debugging identifiers

Not included in `v0.1.1`:

- Cross-sheet joins
- Combined multi-sheet analysis
- Advanced statistics
- Causal inference
- Production-ready object storage
- Persistent session backends

## Validation

Release validation was run with:

- `pytest -q apps/api`
- `cd apps/web && npm run test`
- `cd apps/web && npm run typecheck`
- `cd apps/web && npm run lint`
