# Talk2Sheet v0.3.2

Talk2Sheet `v0.3.2` is a batch workbook analysis release.

This release adds multi-sheet batch execution in one request (still sheet-by-sheet, no cross-sheet join), with real-time progress visibility, partial-failure isolation, and smoother batch selection UX.

## Highlights

- Added batch analysis APIs:
  - `POST /api/spreadsheet/batch`
  - `POST /api/spreadsheet/batch/stream`
- Added per-sheet failure isolation in batch runs:
  - successful sheets still return results
  - failed sheets return `status/error/reason_code`
- Added real progress events and frontend progress bar updates:
  - `batch_progress`
  - `batch_result`
  - `batch_error`
  - `batch_done`
- Added configurable constrained parallel execution for batch tasks:
  - `TALK2SHEET_BATCH_MAX_PARALLEL` (default `1`, hard cap `3`)
- Added batch summary rendering and CSV export in conversation cards
- Improved batch sheet selection UX:
  - invert selection
  - reuse recent selection
  - recent selection hint
- Added batch regression coverage and CI baseline:
  - batch API/stream tests
  - batch performance baseline script and CI artifact upload

## Included In This Release

### Batch execution and streaming

- Added unified batch request/response schemas for workbook-level multi-sheet analysis
- Added normal and SSE batch endpoints with stable event contracts
- Added backend batch pipeline that preserves result ordering while supporting constrained parallel runs
- Added per-sheet error isolation to avoid all-or-nothing batch failures

### Frontend batch experience

- Added real-time batch progress rendering from streaming events
- Added batch result summary table and CSV export support
- Added sheet selection polish: invert, reuse recent selection, and selection persistence improvements

### Quality and CI governance

- Added dedicated batch regression tests, including:
  - success/partial-failure/all-failure scenarios
  - stream progress events
  - constrained-parallel order stability
- Added offline batch performance baseline evaluator and report snapshots
- Added CI layered check for batch performance baseline and artifact upload

## Scope Reminder

Supported in `v0.3.2`:

- Workbook-aware routing and batch orchestration across selected sheets
- Sheet-by-sheet analysis in one batch request with progress visibility
- Partial failure isolation and per-sheet status reporting
- Compare/ranking/share/detail/trend/basic chart/lightweight forecast per sheet
- Clarification and follow-up conversation

Not included in `v0.3.2`:

- Cross-sheet joins in one step
- Unrestricted combined multi-sheet execution
- Advanced statistics and causal inference
- Production-ready object storage and persistent session backends
